# 一、文档说明

本文档明确空间业务的核心功能、角色权限及操作约束，用于指导开发、测试及产品对接，确保各角色操作逻辑清晰、权限边界明确。

# 二、核心角色及整体规则

核心角色分为两类：管理员、普通用户；所有空间操作均关联「空间成员表」，空间相关增删改查需同步维护空间成员表数据；空间内内容可见性：空间内所有成员可查看空间ID对应的内容，仅/editor/admin角色可编辑照片。

# 三、管理员角色功能需求

## 3.1 空间新增

- 操作权限：无限制，可随意新增空间。
- 关联操作：新增空间后，自动在「空间成员表」中添加该管理员为成员，默认配置：spaceRole=admin、maxSize=100、maxCount=100000。
- 约束条件：无额外约束，仅需保证空间ID唯一（默认逻辑，无需额外说明）。

## 3.2 空间删除

- 操作权限：无限制，可随意删除任意空间。
- 关联操作：删除空间时，必须同步删除「空间成员表」中该空间对应的所有成员信息，避免数据残留。
- 约束条件：无（无需判断是否为创建者）。

## 3.3 空间修改

- 操作权限：无限制，可修改空间除ID外的所有字段（如spaceName、spaceType等）。
- 约束条件：禁止修改空间ID，其余字段可随意修改。

## 3.4 空间查询

- 操作权限：无限制，可查询所有空间的全部信息，无权限过滤。
- 关联功能：通过任意空间，可查询该空间下的所有用户信息（即「空间成员表」中该空间对应的所有成员）。

## 3.5 空间成员管理

- 踢出空间：可踢出该空间内除「空间创建者」以外的任意成员。
- 拉人进空间：可随意将用户拉入任意空间，拉入后同步更新「空间成员表」，默认配置遵循空间原有规则（maxSize、maxCount不随拉人操作变更）。

# 四、普通用户角色功能需求

## 4.1 空间新增

- 操作权限：有限制，单个普通用户最多可新增2个空间。
- 关联操作：新增空间后，自动在「空间成员表」中添加该普通用户为成员，默认配置：spaceRole=admin（创建者默认管理员）、maxSize=100、maxCount=100000。
- 约束条件：超出2个空间后，禁止新增，提示“普通用户最多可创建2个空间”。

## 4.2 空间删除

- 操作权限：有限制，仅能删除自己创建的空间。
- 关联操作：删除空间时，必须同步删除「空间成员表」中该空间对应的所有成员信息，避免数据残留。
- 约束条件：非自己创建的空间，无删除权限，提示“仅空间创建者可删除该空间”。

## 4.3 空间修改

- 操作权限：有限制，仅能修改自己创建的空间，且仅可修改spaceName（空间名称）和spaceType（空间类型）两个字段。
- 约束条件：禁止修改空间ID，禁止修改除spaceName、spaceType以外的其他字段；非自己创建的空间，无修改权限。

## 4.4 空间查询

- 操作权限：有限制，仅能查询自己创建的空间，以及自己作为成员加入的空间，无法查看其他无关空间。
- 关联功能：通过自己有权限查看的空间，可查询该空间下的所有用户信息（即「空间成员表」中该空间对应的所有成员）。

## 4.5 空间成员管理（仅自己创建空间的管理员权限）

- 踢出空间：仅可踢出自己创建的空间内的成员，且不能踢出自己（创建者）。
- 拉人进空间：仅可给自己创建的空间拉人，拉入后同步更新「空间成员表」，默认配置遵循空间原有规则。
- 约束条件：非自己创建的空间，无成员管理权限（无法踢人、拉人）。

# 五、通用约束

- 所有空间操作（增删改）均需同步维护「空间成员表」，确保数据一致性。
- 空间内内容可见性：所有加入该空间的成员，均可查看该空间ID对应的内容；仅/editor/admin角色可编辑空间内的照片，其他角色无照片编辑权限。
- 空间ID为唯一标识，所有操作均不可修改空间ID。
