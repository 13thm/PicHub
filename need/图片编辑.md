# 图片协同编辑功能 - API 文档

## 新增文件目录结构

```
src/main/java/com/thm/pichub/
└── websocket/

    ├── EditLock.java            # 编辑锁实体类
    ├── EditLockManager.java     # 编辑锁管理器（控制图片互斥编辑）
    ├── EditLockVO.java          # 编辑锁状态响应
    ├── ImageEditController.java # 图片编辑控制器（HTTP 接口）
    ├── ImageRotateUtil.java     # 图片旋转工具类
    ├── WebSocketConfig.java     # WebSocket 配置类
    ├── WebSocketHandler.java    # WebSocket 处理器（连接、断开、消息接收）
    ├── WsMessage.java           # WebSocket 消息实体类

pom.xml  # 新增依赖：websocket、thumbnailator
```

## WebSocket 消息协议

### 客户端 → 服务端 (WsRequest)

| 字段 | 类型 | 说明 |
|------|------|------|
| action | String | 指令类型：join/leave/edit/rotate/save |
| userId | Long | 用户ID |
| userName | String | 用户名 |
| imageId | Long | 图片ID |
| direction | String | 旋转方向（仅rotate操作）：left/right |

#### 指令类型说明

1. **join** - 加入图片频道
   ```json
   {
     "action": "join",
     "userId": 1,
     "userName": "张三",
     "imageId": 100
   }
   ```

2. **leave** - 离开频道
   ```json
   {
     "action": "leave",
     "userId": 1,
     "userName": "张三",
     "imageId": 100
   }
   ```

3. **edit** - 请求编辑图片
   ```json
   {
     "action": "edit",
     "userId": 1,
     "userName": "张三",
     "imageId": 100
   }
   ```

4. **rotate** - 旋转图片
   ```json
   {
     "action": "rotate",
     "userId": 1,
     "userName": "张三",
     "imageId": 100,
     "direction": "right"  // left:左旋90度, right:右旋90度
   }
   ```

5. **save** - 保存并退出编辑
   ```json
   {
     "action": "save",
     "userId": 1,
     "userName": "张三",
     "imageId": 100
   }
   ```

### 服务端 → 客户端 (WsMessage)

| 字段 | 类型 | 说明 |
|------|------|------|
| type | String | 消息类型 |
| userId | Long | 用户ID |
| userName | String | 用户名 |
| imageId | Long | 图片ID |
| angle | Integer | 旋转角度（ROTATE 消息） |
| message | String | 消息内容 |
| timestamp | Long | 时间戳 |

#### 消息类型说明

1. **JOIN** - 用户加入频道
   ```json
   {
     "type": "JOIN",
     "userId": 1,
     "userName": "张三",
     "imageId": 100,
     "message": "用户加入",
     "timestamp": 1617123456789
   }
   ```

2. **LEAVE** - 用户离开频道
   ```json
   {
     "type": "LEAVE",
     "userId": 1,
     "userName": "张三",
     "imageId": 100,
     "message": "用户离开",
     "timestamp": 1617123456789
   }
   ```

3. **START_EDIT** - 开始编辑
   ```json
   {
     "type": "START_EDIT",
     "userId": 1,
     "userName": "张三",
     "imageId": 100,
     "message": "用户开始编辑",
     "timestamp": 1617123456789
   }
   ```

4. **ROTATE** - 旋转同步
   ```json
   {
     "type": "ROTATE",
     "imageId": 100,
     "angle": 90,
     "message": "图片旋转同步",
     "timestamp": 1617123456789
   }
   ```

5. **SAVE** - 保存成功
   ```json
   {
     "type": "SAVE",
     "userId": 1,
     "userName": "张三",
     "imageId": 100,
     "message": "图片保存成功",
     "timestamp": 1617123456789
   }
   ```

6. **REJECT** - 编辑被拒绝
   ```json
   {
     "type": "REJECT",
     "imageId": 100,
     "message": "当前图片正在编辑中",
     "timestamp": 1617123456789
   }
   ```

7. **AUTO_UNLOCK** - 异常断开解锁
   ```json
   {
     "type": "AUTO_UNLOCK",
     "imageId": 100,
     "message": "编辑者异常断开，编辑锁已释放",
     "timestamp": 1617123456789
   }
   ```

## HTTP 接口

### 1. 保存编辑后的图片

**接口地址**: `POST /api/image/edit/save`

**请求参数**:
- Content-Type: multipart/form-data
- `imageId`: Long - 图片ID
- `file`: MultipartFile - 旋转后的图片文件

**响应示例**:
```json
{
  "code": 0,
  "data": true,
  "message": "ok"
}
```

---

### 2. 获取图片编辑状态

**接口地址**: `GET /api/image/edit/status/{imageId}`

**路径参数**:
- `imageId`: Long - 图片ID

**响应示例**:
```json
{
  "code": 0,
  "data": {
    "imageId": 100,
    "editing": true,
    "editorId": 1,
    "editorName": "张三",
    "angle": 90
  },
  "message": "ok"
}
```

**响应字段说明**:
| 字段 | 类型 | 说明 |
|------|------|------|
| imageId | Long | 图片ID |
| editing | boolean | 是否正在编辑 |
| editorId | Long | 编辑者ID（editing=false时为null） |
| editorName | String | 编辑者名称（editing=false时为null） |
| angle | Integer | 当前旋转角度：0/90/180/270 |

---

### 3. 获取图片当前旋转角度

**接口地址**: `GET /api/image/edit/current-angle/{imageId}`

**路径参数**:
- `imageId`: Long - 图片ID

**响应示例**:
```json
{
  "code": 0,
  "data": 90,
  "message": "ok"
}
```

## 核心功能说明

### 1. 频道机制

- 按图片ID创建独立频道
- 用户查看图片时加入对应频道
- 用户离开时自动离开频道
- 消息仅广播给同一频道的用户

### 2. 编辑互斥锁

- 同一图片同一时间仅允许一个用户编辑
- 编辑者断开连接时自动释放锁
- 其他用户尝试编辑收到拒绝消息

### 3. 实时旋转同步

- 编辑者旋转操作实时同步给所有查看该图片的用户
- 旋转角度存储在内存中（0/90/180/270）
- 前端通过 CSS transform 实现实时预览

### 4. 图片保存

- 编辑者点击保存后，将旋转后的图片上传到 MinIO
- 保存成功后释放编辑锁
- 重置旋转角度到 0 度（因为图片已旋转保存）

## 使用流程示例

### 场景：多用户协同编辑图片

1. **用户A 加入图片100频道**
   ```
   发送: {"action":"join", "userId":1, "userName":"A", "imageId":100}
   收到: {"type":"JOIN", "userId":1, "userName":"A", "imageId":100, ...}
   ```

2. **用户B 加入图片100频道**
   ```
   发送: {"action":"join", "userId":2, "userName":"B", "imageId":100}
   收到: {"type":"JOIN", "userId":2, "userName":"B", "imageId":100, ...}
   ```

3. **用户A 请求编辑图片**
   ```
   发送: {"action":"edit", "userId":1, "userName":"A", "imageId":100}
   收到: {"type":"START_EDIT", "userId":1, "userName":"A", "imageId":100, ...}
   用户B也收到: {"type":"START_EDIT", "userId":1, "userName":"A", "imageId":100, ...}
   ```

4. **用户A 右旋图片90度**
   ```
   发送: {"action":"rotate", "userId":1, "userName":"A", "imageId":100, "direction":"right"}
   收到: {"type":"ROTATE", "imageId":100, "angle":90, ...}
   用户B也看到图片旋转
   ```

5. **用户B 尝试编辑（被拒绝）**
   ```
   发送: {"action":"edit", "userId":2, "userName":"B", "imageId":100}
   收到: {"type":"REJECT", "imageId":100, "message":"当前图片正在编辑中"}
   ```

6. **用户A 保存图片**
   ```
   发送: {"action":"save", "userId":1, "userName":"A", "imageId":100}
   收到: {"type":"SAVE", "userId":1, "userName":"A", "imageId":100, ...}
   编辑锁释放，B现在可以编辑了
   ```

7. **用户A 离开频道**
   ```
   发送: {"action":"leave", "userId":1, "userName":"A", "imageId":100}
   收到: {"type":"LEAVE", "userId":1, "userName":"A", "imageId":100, ...}
   ```
