# 图片协同编辑功能 - 前端需求文档

## 一、功能概述

实现一个多图片协同编辑与实时预览的前端界面，支持多用户同时在线查看、单图片同一时间仅一人可编辑、所有在线用户实时查看操作、系统消息实时广播。

## 二、核心功能需求

### 1. 图片列表页面

**功能说明：** 展示系统内所有可用的图片列表，用户可以选择查看/编辑图片。

**页面元素：**
- 图片缩略图展示（网格布局）
- 图片名称、简介简略信息
- 每张图片显示当前编辑状态：
  - 未编辑：显示「进入编辑」按钮（可点击）
  - 编辑中：显示「编辑中 + 编辑者名称」+ 灰色按钮（不可点击）
- 点击图片进入详情查看/编辑页面

**状态标识：**
- 🔻 红色圆点：图片正在被占用编辑
- 🟢 绿色：图片可编辑

### 2. 图片编辑页面（核心）

**功能说明：** 单张图片的查看与编辑页面，支持实时预览、旋转操作、消息展示。

#### 2.1 主区域 - 图片展示

**功能需求：**
- 居中显示当前图片
- 支持实时旋转预览（通过 CSS transform 实现，无需重新加载图片）
- 显示图片基本信息：
  - 图片名称
  - 图片尺寸
  - 当前旋转角度

**样式要求：**
```css
/* 图片容器 */
.image-container {
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 400px;
  background: #f5f5f5;
  border-radius: 8px;
  overflow: hidden;
}

/* 图片 */
.image-container img {
  max-width: 100%;
  max-height: 500px;
  transition: transform 0.3s ease;
}
```

#### 2.2 操作按钮区域

**功能需求：**

| 按钮 | 显示条件 | 操作 |
|------|---------|------|
| 「进入编辑模式」 | 当前用户不是编辑者且图片未被占用 | 发送 edit 指令 |
| 「左旋 90°」 | 当前用户是编辑者 | 发送 rotate left 指令 |
| 「右旋 90°」 | 当前用户是编辑者 | 发送 rotate right 指令 |
| 「保存」 | 当前用户是编辑者 | 调用 HTTP 接口保存图片 + 发送 save 指令 |
| 「退出编辑」 | 当前用户是编辑者 | 发送 leave 指令，离开编辑模式 |
| 「返回列表」 | 任何情况 | 返回图片列表页 |

**按钮状态设计：**
- 正常状态：蓝色，可点击
- 禁用状态：灰色，不可点击（当前用户不是编辑者时，旋转/保存按钮禁用）
- 编辑中但不是本人：编辑按钮显示「编辑中 - [编辑者名]」并禁用

#### 2.3 消息区域（右侧或底部）

**功能需求：**
- 实时显示系统消息
- 消息类型与展示样式：

| 消息类型 | 图标 | 颜色 | 示例 |
|---------|------|------|------|
| JOIN | 👋 | 蓝色 | 👋 张三 加入了查看 |
| LEAVE | 🚪 | 灰色 | 🚪 张三 离开了 |
| START_EDIT | ✏️ | 黄色 | ✏️ 张三 开始编辑 |
| ROTATE | 🔄 | 绿色 | 🔄 图片向右旋转 90 度 |
| SAVE | 💾 | 蓝色 | 💾 张三 保存了图片 |
| REJECT | ❌ | 红色 | ❌ 当前图片正在编辑中 |
| AUTO_UNLOCK | ⚠️ | 橙色 | ⚠️ 编辑者异常断开 |

**消息列表要求：**
- 消息按时间倒序排列（最新在上）
- 限制显示最近 50 条消息
- 可滚动查看历史消息
- 每条消息显示时间戳（如：15:30:25）

#### 2.4 在线用户显示

**功能需求：**
- 显示当前正在查看此图片的用户列表
- 标记当前用户
- 标记当前编辑者（如有）

**展示样式：**
```
👤 在线用户 (3)
  • 张三（我）
  • 李四
  • 王五 ✏️
```

### 3. WebSocket 连接管理

#### 3.1 连接生命周期

```javascript
// 页面加载时建立连接
componentDidMount() {
  connectWebSocket();
}

// 获取到用户信息和图片ID后，加入频道
onDataLoaded(userId, userName, imageId) {
  sendJoinMessage(userId, userName, imageId);
}

// 组件卸载时离开频道
componentWillUnmount() {
  sendLeaveMessage();
  disconnectWebSocket();
}
```

#### 3.2 连接状态处理

| 状态 | 处理方式 |
|------|---------|
| 连接成功 | 更新 UI 显示"已连接" |
| 连接失败 | 显示重连按钮 |
| 连接断开 | 显示"连接已断开，正在重连..."并自动重试 |
| 重连成功 | 重新加入当前频道 |

#### 3.3 心跳保活

```javascript
// 每隔 30 秒发送一次心跳
setInterval(() => {
  if (ws.readyState === WebSocket.OPEN) {
    ws.send(JSON.stringify({ type: "ping" }));
  }
}, 30000);
```

#### 3.4 重连机制

- 连接断开后自动重连
- 每次重连间隔：1秒、2秒、5秒、10秒（指数退避）
- 最大重试次数：无限制（持续重试）
- 重连成功后，重新发送 join 消息恢复状态

### 5. 消息处理逻辑

#### 5.1 消息接收与处理

```typescript
const handleMessage = (message: WsMessage) => {
  switch (message.type) {
    case 'JOIN':
      // 添加在线用户
      // 添加消息到列表
      addMessage('系统消息', `${message.userName} 加入了查看`);
      break;

    case 'LEAVE':
      // 移除在线用户
      addMessage('系统消息', `${message.userName} 离开了`);
      break;

    case 'START_EDIT':
      // 设置编辑状态
      setEditing(true, message.editorId, message.editorName);
      addMessage('系统消息', `${message.editorName} 开始编辑`);
      break;

    case 'ROTATE':
      // 更新旋转角度
      setRotationAngle(message.angle);
      addMessage('编辑操作', `图片旋转到 ${message.angle} 度`);
      break;

    case 'SAVE':
      // 释放编辑锁
      clearEditing();
      setRotationAngle(0);
      addMessage('完成', `${message.editorName} 保存了图片`);
      break;

    case 'REJECT':
      // 显示错误提示
      showError('当前图片正在编辑中，无法进入编辑模式');
      break;

    case 'AUTO_UNLOCK':
      // 释放编辑锁
      clearEditing();
      addMessage('警告', '编辑者异常断开，编辑锁已释放');
      break;
  }
};
```

#### 5.2 消息限制

- 最多保留 50 条消息
- 新消息追加到列表顶部
- 超过限制时移除最旧的消息

### 6. 用户交互流程

### 6.1 进入编辑模式流程

```
用户点击「进入编辑」
    ↓
检查：连接是否正常？
    ↓ 否
显示错误「连接已断开，请刷新页面」
    ↓ 是
发送 { action: "edit", userId, userName, imageId }
    ↓
等待服务器响应
    ↓
收到 START_EDIT → 进入编辑模式，显示旋转按钮
收到 REJECT → 显示错误「当前图片正在编辑中」
```

### 6.2 旋转操作流程

```
用户点击「左旋」或「右旋」
    ↓
检查：当前用户是否是编辑者？
    ↓ 否
提示「您不是此图片的编辑者」
    ↓ 是
计算新角度
    ↓
更新本地 UI（立即预览）
    ↓
发送 { action: "rotate", userId, userName, imageId, direction }
    ↓
后端广播 ROTATE 消息，其他用户看到旋转
```

### 6.3 保存流程

```
用户点击「保存」
    ↓
检查：是否有修改？（角度是否为 0）
    ↓ 无
提示「无需保存」
    ↓ 有
显示保存中提示
    ↓
执行图片旋转处理
    ↓
调用 POST /api/image/edit/save 上传旋转后的图片
    ↓
保存成功？
    ↓ 否
显示错误「保存失败」
    ↓ 是
发送 { action: "save", userId, userName, imageId }
    ↓
收到 SAVE 消息 → 退出编辑模式
```

### 6.4 切换图片流程

```
用户返回列表 → 选择另一张图片
    ↓
发送 { action: "leave", userId, userName, currentImageId }
    ↓
清除当前频道状态
    ↓
发送 { action: "join", userId, userName, newImageId }
    ↓
加载新图片
    ↓
发送 GET /api/image/edit/status/{newImageId} 获取状态
    ↓
根据返回数据初始化状态（是否被编辑、当前角度等）
```

### 7. 图片保存 - 前端图片处理

#### 7.1 图片旋转处理

使用 Canvas 或第三方库（如 `react-image-crop`）在前端实现图片旋转：

```typescript
// 使用 Canvas 旋转图片
const rotateImage = async (imageSrc: string, angle: number): Promise<Blob> => {
  return new Promise((resolve, reject) => {
    const img = new Image();
    img.crossOrigin = 'anonymous';
    img.onload = () => {
      const canvas = document.createElement('canvas');
      const ctx = canvas.getContext('2d');

      // 根据旋转角度调整画布尺寸
      if (angle === 90 || angle === 270) {
        canvas.width = img.height;
        canvas.height = img.width;
      } else {
        canvas.width = img.width;
        canvas.height = img.height;
      }

      // 旋转画布
      ctx.translate(canvas.width / 2, canvas.height / 2);
      ctx.rotate(angle * Math.PI / 180);

      // 绘制图片
      ctx.drawImage(
        img,
        -img.width / 2,
        -img.height / 2,
        img.width,
        img.height
      );

      canvas.toBlob((blob) => {
        if (blob) {
          resolve(blob);
        } else {
          reject(new Error('Failed to create blob'));
        }
      }, 'image/jpeg', 0.9);
    };

    img.onerror = () => reject(new Error('Failed to load image'));
    img.src = imageSrc;
  });
};
```

#### 7.2 保存接口调用

```typescript
const saveImage = async (imageId: number, rotatedBlob: Blob) => {
  const formData = new FormData();
  formData.append('imageId', imageId.toString());
  formData.append('file', rotatedBlob, 'rotated-image.jpg');

  try {
    const response = await fetch('/api/image/edit/save', {
      method: 'POST',
      body: formData,
    });

    if (!response.ok) {
      throw new Error('保存失败');
    }

    const result = await response.json();
    return result.data;
  } catch (error) {
    console.error('保存图片失败', error);
    throw error;
  }
};
```

### 16. 附录：TypeScript 类型定义

```typescript
// WebSocket 消息类型
type WsMessageType =
  | 'JOIN'
  | 'LEAVE'
  | 'START_EDIT'
  | 'ROTATE'
  | 'SAVE'
  | 'REJECT'
  | 'AUTO_UNLOCK';

// WebSocket 消息
interface WsMessage {
  type: WsMessageType;
  userId?: number;
  userName?: string;
  imageId: number;
  angle?: number;
  message: string;
  timestamp: number;
}

// WebSocket 请求
interface WsRequest {
  action: 'join' | 'leave' | 'edit' | 'rotate' | 'save';
  userId: number;
  userName: string;
  imageId: number;
  direction?: 'left' | 'right';
}

// 编辑锁状态
interface EditLockStatus {
  imageId: number;
  editing: boolean;
  editorId?: number;
  editorName?: string;
  angle: number;
}

// 用户信息
interface User {
  id: number;
  name: string;
  isMe: boolean;
  isEditor: boolean;
}

// 消息项
interface Message {
  id: string;
  type: WsMessageType;
  content: string;
  timestamp: number;
}
```
